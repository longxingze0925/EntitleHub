# 数据库表结构设计

本文档定义后台系统第一版数据库结构。数据库推荐使用 PostgreSQL。所有表结构均服务于以下核心目标：

- 多租户隔离。
- 后台用户和客户用户分离。
- 支持软件授权、订阅、设备绑定。
- 支持客户端 session、refresh token 轮换和撤销。
- 支持版本包、热更新、脚本安全分发。
- 支持审计、限流、任务和系统配置。

## 1. 通用规范

### 1.1 ID 规范

所有主键使用 UUID。

```sql
id uuid primary key
```

建议由应用层生成 UUID v4 或 UUID v7。若使用 PostgreSQL 生成，需要启用扩展。

### 1.2 时间字段

所有业务表默认包含：

```sql
created_at timestamptz not null default now()
updated_at timestamptz not null default now()
deleted_at timestamptz null
```

软删除表必须所有查询默认排除 `deleted_at is not null` 的记录。

### 1.3 多租户字段

所有租户业务表必须包含：

```sql
tenant_id uuid not null
```

平台级表除外，例如系统配置、全局迁移记录等。

### 1.4 状态字段

状态字段使用 text 或 enum。第一版建议使用 text，并在业务层校验，避免频繁改 enum 迁移。

### 1.5 索引原则

所有高频查询必须建索引：

- `tenant_id`
- `app_id`
- `customer_id`
- `device_id`
- `session_id`
- `expires_at`
- `revoked_at`
- `status`

复合索引优先按查询条件顺序创建。

## 3. 租户表 tenants

用途：系统最高业务隔离单位。

```sql
create table tenants (
  id uuid primary key,
  name text not null,
  slug text not null unique,
  status text not null default 'active',
  plan text not null default 'free',
  max_applications int not null default 3,
  max_team_members int not null default 5,
  max_customers int not null default 1000,
  metadata jsonb not null default '{}'::jsonb,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  deleted_at timestamptz null
);
```

状态：

```text
active
suspended
deleted
```

索引：

```sql
create unique index idx_tenants_slug on tenants(slug) where deleted_at is null;
create index idx_tenants_status on tenants(status);
```

## 4. 后台成员表 team_members

用途：管理后台登录用户。

```sql
create table team_members (
  id uuid primary key,
  tenant_id uuid not null references tenants(id),
  email text not null,
  password_hash text not null,
  name text not null,
  phone text null,
  avatar text null,
  status text not null default 'active',
  email_verified boolean not null default false,
  mfa_enabled boolean not null default false,
  mfa_secret_encrypted text null,
  last_login_at timestamptz null,
  last_login_ip inet null,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  deleted_at timestamptz null
);
```

状态：

```text
active
disabled
invited
```

索引：

```sql
create unique index idx_team_members_tenant_email
on team_members(tenant_id, lower(email))
where deleted_at is null;

create index idx_team_members_tenant_status
on team_members(tenant_id, status);
```

注意：

- `password_hash` 必须是 Argon2id。
- `mfa_secret_encrypted` 必须加密存储。
- 禁止返回 `password_hash` 和 `mfa_secret_encrypted`。

### 4.1 MFA 恢复码表 admin_mfa_recovery_codes

用途：保存管理员 MFA 恢复码的 hash。恢复码只展示一次，不能明文入库。

```sql
create table admin_mfa_recovery_codes (
  id uuid primary key,
  tenant_id uuid not null references tenants(id),
  team_member_id uuid not null references team_members(id),
  code_hash text not null,
  used_at timestamptz null,
  revoked_at timestamptz null,
  created_at timestamptz not null default now()
);
```

索引：

```sql
create index idx_admin_mfa_recovery_codes_member
on admin_mfa_recovery_codes(team_member_id, used_at, revoked_at);
```

规则：

- 恢复码只在创建或重新生成时返回一次。
- 数据库只存 hash。
- 使用后必须写入 `used_at`。
- 重新生成恢复码时，旧恢复码必须全部写入 `revoked_at`。

### 4.2 一次性 token 表 one_time_tokens

用途：统一保存邀请、邮箱验证、密码重置等短期 token。

```sql
create table one_time_tokens (
  id uuid primary key,
  tenant_id uuid null references tenants(id),
  purpose text not null,
  subject_type text not null,
  subject_id uuid null,
  email text null,
  token_hash text not null unique,
  created_by uuid null references team_members(id),
  expires_at timestamptz not null,
  consumed_at timestamptz null,
  revoked_at timestamptz null,
  metadata jsonb not null default '{}'::jsonb,
  created_at timestamptz not null default now()
);
```

`purpose` 建议值：

```text
team_invite
admin_password_reset
customer_password_reset
email_verify
```

索引：

```sql
create index idx_one_time_tokens_subject
on one_time_tokens(tenant_id, subject_type, subject_id, purpose);

create index idx_one_time_tokens_active
on one_time_tokens(purpose, expires_at)
where consumed_at is null and revoked_at is null;
```

规则：

- token 明文只通过邮件或一次性响应发送。
- 数据库存 `token_hash`，建议使用 `HMAC-SHA256(TOKEN_HASH_PEPPER, token)`。
- token 使用后必须写入 `consumed_at`。
- 过期、已使用、已撤销 token 必须拒绝。
- 接受邀请、重置密码、验证邮箱必须写审计日志。

### 6.2 admin_refresh_tokens

```sql
create table admin_refresh_tokens (
  id uuid primary key,
  session_id uuid not null references admin_sessions(id),
  token_hash text not null unique,
  created_at timestamptz not null default now(),
  expires_at timestamptz not null,
  used_at timestamptz null,
  revoked_at timestamptz null
```

- 旧 token 被复用时撤销整个 session。

## 8.1 签名密钥表 signing_keys

用途：保存服务端用于签发 access token、更新包签名、脚本签名、应用级签名的公私钥元数据。

```sql
create table signing_keys (
  id uuid primary key,
  tenant_id uuid null references tenants(id),
  app_id uuid null references applications(id),
  key_scope text not null,
  kid text not null unique,
  alg text not null default 'EdDSA',
  public_key_pem text not null,
  private_key_envelope jsonb null,
  status text not null default 'active',
  not_before timestamptz not null default now(),
  not_after timestamptz null,
  rotated_from_id uuid null references signing_keys(id),
  created_by uuid null references team_members(id),
  created_at timestamptz not null default now(),
  activated_at timestamptz null,
  retired_at timestamptz null,
  revoked_at timestamptz null
);
```

`key_scope` 建议值：

```text
jwt_access_token
release_file
secure_script
app_request
```

状态：

```text
active
retiring
retired
revoked
```

索引：

```sql
create index idx_signing_keys_scope_status
on signing_keys(key_scope, status, not_before, not_after);

create index idx_signing_keys_app_scope
on signing_keys(tenant_id, app_id, key_scope, status);
```

规则：

- `kid` 必须全局唯一，所有签名结果必须带 `kid`。
- `jwt_access_token` 使用全局 key，`tenant_id` 和 `app_id` 为 `null`。
- JWKS 只暴露 public key，不暴露 private key。
- `private_key_envelope` 必须使用 `MASTER_KEY` 或 KMS 加密保存私钥。
- `private_key_envelope` 至少包含 `ciphertext`、`nonce`、`tag`、`key_version`，使用 KMS 时还要包含 `kms_key_id`。
- 轮换时新 key 先 `active`，旧 key 进入 `retiring`，等旧 token 或旧包过期后再 `retired`。
- 全局 `jwt_access_token` key 通过后台安全接口轮换；旧 key 进入 `retiring` 时写入 `not_after = now + CLIENT_ACCESS_TOKEN_TTL_SECONDS`，`active` 和未过 `not_after` 的 `retiring` key 会进入公开 `/.well-known/jwks.json`。
- `revoked` key 不能再用于签名，但可保留用于审计。

```sql

  id uuid primary key,
  tenant_id uuid not null references tenants(id),
  app_id uuid not null references applications(id),
  device_id uuid not null references devices(id),
```


```text
active
```

```sql
  signature_alg text not null,
  required_features jsonb not null default '[]'::jsonb,
  expires_at timestamptz null,
  published_at timestamptz null,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  deleted_at timestamptz null
);
```

索引：

```sql
create index idx_secure_scripts_app_status
on secure_scripts(tenant_id, app_id, status, version_code desc);
```

规则：

- `content_ciphertext` 是服务端加密存储字段，使用 `MASTER_KEY` 或 KMS 保护。
- 客户端拉取脚本时，服务端解密后返回 `content_base64`。
- 客户端不负责解密，只负责 hash 和签名校验。
- 如果后续要做端到端加密，必须单独设计客户端密钥分发，不要复用 `MASTER_KEY`。

## 18. 下载 token 表 download_tokens

如果下载 token 使用 JWT 且无需撤销，可以不建表。若需要可撤销短链，建表：

```sql
create table download_tokens (
  id uuid primary key,
  tenant_id uuid not null references tenants(id),
  app_id uuid not null references applications(id),
  device_id uuid not null references devices(id),
  file_id uuid not null,
  token_hash text not null unique,
  kind text not null,
  expires_at timestamptz not null,
  used_at timestamptz null,
  revoked_at timestamptz null,
  created_at timestamptz not null default now()
);
```

建议第一版使用 JWT 短时下载 token，后续需要单次使用时再落表。

如果下载 token 落表：

- token 明文只返回一次。
- `token_hash` 建议使用 `HMAC-SHA256(TOKEN_HASH_PEPPER, token)`。
- token 必须绑定 `tenant_id`、`app_id`、`device_id`、`file_id`。

## 23. 通知渠道 notification_channels

用于后台页面配置 webhook、SMTP 和值班系统通知渠道。敏感字段加密保存，列表接口不回显明文。

```sql
create table notification_channels (
  id uuid primary key default gen_random_uuid(),
  tenant_id uuid not null references tenants(id) on delete cascade,
  name text not null,
  kind text not null,
  enabled boolean not null default true,
  config_json jsonb not null default '{}'::jsonb,
  secret_encrypted text null,
  last_test_status text null,
  last_test_error text null,
  last_test_at timestamptz null,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);
```

约束：

```sql
kind in ('webhook', 'email', 'pagerduty')
jsonb_typeof(config_json) = 'object'
last_test_status is null or last_test_status in ('success', 'failed')
```

索引：

```sql
create unique index idx_notification_channels_tenant_name
on notification_channels(tenant_id, lower(name));

create index idx_notification_channels_tenant_kind
on notification_channels(tenant_id, kind);
```

安全要求：

- `config_json` 只放非敏感配置和 `target_summary`。
- `secret_encrypted` 存放经 `MASTER_KEY` envelope 加密后的 JSON secret。
- webhook URL、SMTP password、PagerDuty routing key 不允许明文返回或写入审计日志。

## 24. 必须避免的表设计错误

禁止：

- `users` 同时表示后台管理员和客户。
- 授权码明文存储。
- refresh token 明文存储。
- app_secret 明文存储。
- 私钥明文存储。
- 业务表没有 tenant_id。
- 只用 machine_id 作为设备唯一可信身份。
- 审计日志保存敏感 token。
- 删除关键业务数据时硬删除。
