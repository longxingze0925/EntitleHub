# 权限点与错误码清单

本文档用于固定后台权限点和 API 错误码。

作用：

- 后端权限中间件按本文档校验。
- 前端菜单、按钮、路由按本文档显示。
- API 返回错误码按本文档稳定输出。
- AI 开发新接口时，先查本文档，不要临时发明权限点或错误码。

---

## 1. 权限命名规则

格式：

```text
resource:action
```

要求：

- 全小写。
- 使用英文。
- resource 使用业务资源名。
- action 使用明确动作。
- 不使用模糊权限，例如 `admin:all`、`manage:all`。
- 超级管理员能力通过角色聚合，不通过接口跳过权限。

常用 action：

```text
read
create
update
delete
disable
enable
invite
revoke
suspend
renew
publish
deprecate
upload
download
export
rotate_key
blacklist
unblacklist
unbind
reset_password
reset_device
```

## 2. 权限点清单

### 2.1 租户

```text
tenant:read
tenant:update
tenant:delete
```

说明：

- `tenant:delete` 必须只允许 owner 或系统级管理员。
- 删除租户必须软删除，并撤销相关 session。

### 2.2 团队成员

```text
member:read
member:invite
member:update
member:disable
member:enable
member:delete
member:reset_password
```

说明：

- 禁用成员必须撤销其后台 session。
- owner 不能被普通 admin 禁用。
- 邀请成员使用 `one_time_tokens`，token 只存 hash。

### 2.3 角色与权限

```text
role:read
role:create
role:update
role:delete
permission:read
```

说明：

- `permission:*` 第一版只读，不允许后台随意创建系统权限点。
- 内置角色不允许删除。
- 修改角色权限必须写审计日志。

### 2.4 客户

```text
customer:read
customer:create
customer:update
customer:disable
customer:enable
customer:delete
customer:reset_password
```

说明：

- 禁用客户必须撤销其客户端 session。
- 重置客户密码必须让旧 refresh token 失效。
- 删除客户优先软删除。

### 2.5 应用

```text
app:read
app:create
app:update
app:delete
app:rotate_key
app:read_key
```

说明：

- `app:read_key` 只能读取 public key 和 key 元数据。
- 私钥永远不允许通过 API 返回。
- 轮换应用密钥必须写审计日志。

### 2.6 授权

```text
license:read
license:create
license:update
license:revoke
license:suspend
license:renew
license:reset_device
```

说明：

- 授权码数据库只存 hash。
- 吊销授权必须联动撤销相关 client session。
- 重置设备必须记录原因和操作人。

### 2.7 订阅

```text
subscription:read
subscription:create
subscription:update
subscription:cancel
subscription:renew
```

说明：

- 第一版可以不接支付系统。
- 订阅状态仍必须影响客户端授权判断。

### 2.8 设备

```text
device:read
device:update
device:unbind
device:blacklist
device:unblacklist
device:revoke_session
```

说明：

- 拉黑设备必须让 heartbeat、refresh、verify 失败。
- 解绑设备必须撤销对应 client session。
- 设备私钥只在客户端本地，后台不能保存。

### 2.9 版本分发

```text
release:read
release:upload
release:create
release:update
release:publish
release:deprecate
release:delete
```

说明：

- 上传文件必须计算 sha256。
- 发布版本必须签名。
- 发布后不能随意修改文件内容，只能发布新版本。

### 2.10 安全脚本

```text
script:read
script:create
script:update
script:publish
script:deprecate
script:revoke
script:delete
```

说明：

- 脚本属于高危资源。
- 发布脚本必须签名。
- 客户端只负责拉取和验签，不自动执行脚本。

### 2.11 审计

```text
audit:read
audit:export
```

说明：

- 审计日志不允许普通删除。
- 导出审计日志必须记录审计。

### 2.12 系统与安全

```text
system:read
system:update
security:read
security:revoke_session
security:rotate_key
security:view_events
security:retry_event
```

说明：

- `security:rotate_key` 用于签名密钥、应用密钥等敏感轮换。
- 密钥轮换必须记录 `kid`、操作人、旧 key、新 key。

### 2.13 通知渠道

```text
notification:read
notification:update
```

说明：

- `notification:read` 可查看渠道列表和密钥是否已配置。
- `notification:update` 可创建、更新、清空密钥和执行 dry-run 测试。
- 通知渠道接口不得返回明文密钥。

### 2.14 AI 计费

```text
ai:read
ai:provider:update
ai:model:update
ai:wallet:update
ai:api_key:update
ai:asset:delete
```

说明：

- `ai:read` 可查看 AI 渠道、模型价格、客户余额、调用记录、缓存素材和流水。
- `ai:provider:update` 可创建、更新、停用三方渠道和密钥。
- `ai:model:update` 可创建、更新、停用对外模型、销售价格和模型每日限额。
- `ai:wallet:update` 可手动充值、扣减客户 AI 钱包余额，或更新客户钱包每日限额。
- `ai:api_key:update` 可为客户创建、更新或吊销 AI 网关 API Key。
- `ai:asset:delete` 可软删除 AI 生成缓存素材。
- 三方密钥不得返回前端明文，余额调整必须写审计和账本流水。

### 2.15 服务端密钥

```text
server_api_key:read
server_api_key:update
```

说明：

- `server_api_key:read` 可查看 Server Key 列表、展示前缀、绑定应用、状态和最后使用时间。
- `server_api_key:update` 可创建、更新、吊销 Server Key。
- Server Key 明文只在创建时返回一次，不允许通过查询接口再次返回。
- Server Key 只能由可信业务后端保存，浏览器和客户端不得持有。

## 4. 错误码规则

返回格式固定：

```json
{
  "code": 40100,
  "message": "unauthenticated",
  "data": null,
  "request_id": "req_xxx"
}
```

要求：

- `code` 给程序判断。
- `message` 给开发排查，不直接当最终用户文案。
- 前端展示文案可以按 `code` 映射。
- 不要把密码、token、签名、私钥放进错误信息。

## 6. 使用要求

后端：

- 每个受保护接口必须绑定一个或多个权限点。
- 权限点不存在时，启动或测试阶段应失败。
- 错误码必须通过统一 error 类型返回。
- 不允许 handler 里随手写数字错误码。

前端：

- 菜单显示根据 permissions。
- 按钮显示根据 permissions。
- 但是安全判断必须以后端为准。
- 错误提示根据 code 映射，不要只匹配 message。

测试：

- 每个权限点至少有一个 403 测试。
- 每个认证错误至少有一个 401 测试。
- refresh token 复用必须返回 `40108`。
- 越租户访问必须返回 `40301` 或 `40400`，不能返回对方真实数据。

## 7. 新增权限和错误码流程

新增接口时：

1. 先确认是否能复用已有权限点。
2. 不能复用时，在本文档新增权限点。
3. 在数据库 seed 中新增 permission。
4. 在 API 文档写明权限要求。
5. 在前端权限映射中加入该权限点。
6. 补 401、403、正常访问测试。

新增错误码时：

1. 先确认是否能复用已有错误码。
2. 不能复用时，在本文档新增错误码。
3. 在后端 error enum 中新增。
4. 在前端错误文案映射中新增。
5. 补对应测试。
